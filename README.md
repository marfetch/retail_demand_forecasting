# Retail Demand Forecasting

## Краткое описание

Retail Demand Forecasting - pet-project по прогнозированию розничного спроса на Python. Проект построен вокруг дневного временного ряда продаж для магазина `store_nbr = 1` и товарной группы `GROCERY I` из соревнования Kaggle Store Sales - Time Series Forecasting.

Если товарная группа `GROCERY I` отсутствует в данных для магазина 1, код автоматически выбирает товарную группу с максимальной суммарной продажей для этого магазина.

## Цель проекта

Спрогнозировать продажи на следующие 30 дней на основе исторических данных и показать полный аналитический цикл: подготовку данных, EDA, обработку выбросов, декомпозицию временного ряда, проверку стационарности, построение baseline, SARIMA и Prophet-моделей, сравнение качества и бизнес-интерпретацию.

## Бизнес-контекст

Прогноз спроса помогает розничной сети:

- планировать закупки и пополнение запасов;
- снижать риск out-of-stock и избыточных остатков;
- оценивать влияние промо и праздничных дней;
- планировать персонал и операционные ресурсы;
- принимать решения на горизонте ближайшего месяца.

## Методология

1. Загрузка исходных CSV-файлов.
2. Выбор одного интерпретируемого временного ряда: магазин 1 и товарная группа `GROCERY I`.
3. Агрегация дневных продаж по дате.
4. Добавление внешних факторов: промо, праздники, цена нефти, транзакции.
5. Заполнение пропущенных дат.
6. Winsorization выбросов по IQR.
7. Добавление календарных признаков.
8. EDA временного ряда.
9. Декомпозиция ряда на тренд, сезонность и остаток.
10. Проверка стационарности ADF-тестом.
11. Train/test split: последние 30 дней используются как test.
12. Обучение baseline, SARIMA и Prophet.
13. Сравнение качества по нескольким метрикам.
14. Финальный прогноз на 30 дней вперед.

## Метрики качества

В проекте используются:

- **MAE** - средняя абсолютная ошибка;
- **RMSE** - корень из средней квадратичной ошибки;
- **MAPE** - средняя абсолютная процентная ошибка;
- **WAPE** - взвешенная абсолютная процентная ошибка.

Главная метрика проекта - **WAPE**, потому что она хорошо подходит для продаж: показывает ошибку относительно общего объема спроса и устойчивее MAPE при отдельных низких или нулевых значениях продаж.

## Модели

- **Baseline**: среднее значение продаж за последние 7 дней train-периода.
- **SARIMA**: модель `SARIMAX` из `statsmodels` с недельной сезонностью.
- **Prophet**: модель с недельной и годовой сезонностью, а также доступными регрессорами.

Если библиотека `prophet` не установлена, ноутбук `05_prophet_model.ipynb` выводит понятное сообщение и не ломает выполнение остальных ячеек.

## Ограничения проекта

- Для демонстрации выбран один временной ряд, а не все магазины и товарные группы.
- Финальный горизонт прогноза ограничен 30 днями.
- Будущие значения внешних регрессоров для Prophet могут быть неизвестны, поэтому для будущего периода используются простые допущения.
- SARIMA-параметры заданы разумными базовыми значениями без тяжелого перебора.
- Прогноз не учитывает возможные изменения ассортимента, ценовой политики и supply chain constraints.
- Последний месяц исходного ряда (`2017-08`) неполный, поэтому месячные суммы за него нельзя напрямую сравнивать с полными месяцами.
- Kaggle `test.csv` не используется для оценки качества, так как в нем нет фактических `sales`; качество оценивается на последних 30 днях `train.csv`.
- Корреляции в EDA не интерпретируются как причинно-следственные связи.

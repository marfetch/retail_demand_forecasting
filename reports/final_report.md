# Final Report

Актуализировано по файлам `data/processed/model_comparison.csv` и `data/processed/final_30_days_forecast.csv` после повторного запуска ноутбуков: `2026-05-02`.

## Краткое резюме проекта

Retail Demand Forecasting - проект по прогнозированию розничного спроса на 30 дней вперед на основе исторических продаж Kaggle Store Sales - Time Series Forecasting.

Лучшей моделью на тестовом периоде стала `Prophet` с WAPE `9.70%`. Финальный прогноз построен на период `2017-08-16` - `2017-09-14`.

## Что было реализовано

- загрузка исходных данных;
- выбор одного интерпретируемого временного ряда;
- агрегация продаж по дням;
- добавление промо, праздников, транзакций и цены нефти;
- заполнение пропущенных дат;
- обработка выбросов через IQR winsorization;
- EDA;
- декомпозиция временного ряда;
- ADF-тест стационарности;
- baseline-прогноз;
- SARIMA-модель;
- Prophet-модель;
- сравнение моделей по MAE, RMSE, MAPE, WAPE;
- финальный прогноз на 30 дней вперед.

## Стационарность и декомпозиция

ADF p-value исходного ряда:

`0.0036`

ADF p-value после логарифмирования и первого дифференцирования:

`1.32e-28`

Вывод: исходный ряд стационарен по ADF-тесту на уровне 5%, но сезонность по дням недели остается заметной, поэтому SARIMA с недельной сезонной компонентой является обоснованным baseline-подходом статистического моделирования.

## Модели прогнозирования

Использовались:

- Baseline;
- SARIMA;
- Prophet.

SARIMA улучшила baseline по WAPE: `15.38%` против `19.50%`.

Prophet показал лучшую метрику WAPE: `9.70%`. Вероятная причина преимущества - явное моделирование сезонности и использование регрессоров.

## Бизнес-выводы

Прогноз может использоваться для:

- планирования закупок;
- поддержания целевого уровня запасов;
- оценки риска дефицита;
- планирования промо-акций;
- распределения операционных ресурсов магазина.

Суммарный прогноз за 30 дней можно использовать как ориентир для закупки на месяц, а дневной прогноз - для планирования пополнений и контроля риска out-of-stock. WAPE удобен для бизнеса, потому что показывает ошибку относительно общего объема продаж.

## Ограничения

- Анализируется один временной ряд.
- Внешние факторы для будущего периода могут быть неизвестны.
- Для будущего Prophet-прогноза используются допущения по регрессорам.
- Не проводится полный автоматический подбор гиперпараметров.
- Нет rolling backtesting.
- Не учитываются цены, маржинальность и ограничения поставок.
